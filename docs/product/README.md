# MediLocal — Product Documentation

The five foundational product documents, **reverse-documented from the built codebase** (M0–M5
engineering complete) on 2026-07-12. Status tags throughout: **[Built]** / **[Planned]** /
**[Deferred]** and ✅/🟡/⬜. The engineering + architecture source of truth remains
[../PLAN.md](../PLAN.md); these reframe it into standard product artefacts.

1. [PRD](01-PRD.md) — problem, users, vision, features (must/nice), app flow, MVP, metrics, non-goals.
2. [Technical Architecture](02-technical-architecture.md) — tech stack + versions, folder structure, full DB schema, env/config.
3. [Security & Access](03-security-and-access.md) — auth methods, roles/permissions, data-scoping rules, error handling, edge cases.
4. [Frontend Specification](04-frontend-spec.md) — mobile + dashboard design systems (real tokens), API & third-party integration spec.
5. [Feature Ticket List](05-feature-tickets.md) — every feature as a ticket with acceptance criteria, dependencies, priority, and build status.

> These describe what is **built and planned**; they are not a fresh greenfield spec. Where a document
> and the code ever disagree, the code wins — update the doc.
